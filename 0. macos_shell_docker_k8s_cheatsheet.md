# macOS + Docker + Kubernetes 치트시트

> 목적: **macOS(터미널/zsh)**에서 네트워크 점검/설정, 그리고
> **Docker / Kubernetes(kubectl)** 운영 시 자주 쓰는 명령어를
> **실행 예시 + 출력 형태**로 빠르게 참고합니다.

- 날짜: 2026-01-10
- 주의: 출력은 **환경(네트워크, 권한, OS, 클러스터 구성)**에 따라 달라집니다.
  아래는 **형태를 익히는 샘플**입니다.

---

## macOS 초기 설정 (예시, 선택)

VM 중심으로 실습하면 macOS 설정은 최소화해도 됩니다.  
아래는 **macOS에서 직접 kubectl/Docker를 쓸 때만** 필요합니다.

Terminal(zsh):

```bash
# Homebrew (미설치 시)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

brew update

# Docker Desktop 또는 Colima 중 택1
brew install --cask docker
# brew install colima docker

# kubectl
brew install kubectl

# (Apple Silicon) 이미지 아키텍처는 arm64 지원 여부 확인
```

---

## 그러나! 
대부분은 UTM VM 안에서 합니다.
macOS에서 하는 건 최소한으로 보면 아래 정도:

- VM 접속용 SSH 클라이언트
- 브라우저로 대시보드/Ingress 확인
- /etc/hosts에 도메인 매핑
- (선택) macOS에 kubectl 설정해서 로컬에서 관리
- 즉, 실습 대부분은 VM이고 macOS는 “접속/확인용” 역할.

## 1) macOS 네트워크

### 1-1) IP / NIC / 게이트웨이 / DNS 확인

```bash
ifconfig
networksetup -listallhardwareports
ipconfig getifaddr en0   # 보통 Wi-Fi, 환경에 따라 en1/en2일 수 있음
```

예시 출력(일부):
```text
en0: flags=8863<UP,BROADCAST,RUNNING,SIMPLEX,MULTICAST> mtu 1500
	inet 192.168.0.10 netmask 0xffffff00 broadcast 192.168.0.255
```

> `networksetup -listallhardwareports`에서 Wi-Fi/USB Ethernet에 매핑된 장치명(en0/en1 등)을 확인한 뒤 `ipconfig getifaddr`에 사용합니다.

```bash
route -n get default
scutil --dns
networksetup -getdnsservers "Wi-Fi"
```

예시:
```text
gateway: 192.168.0.1
nameserver[0] : 1.1.1.1
nameserver[1] : 8.8.8.8
```

---

### 1-2) DNS 조회 / 검증

```bash

```

예시:
```text

Address: 1.1.1.1

Non-authoritative answer:
Name:    naver.com
Addresses: 223.130.195.200
           223.130.195.95
```

```bash
dig +short naver.com
```

예시:
```text
223.130.195.200
```

---

### 1-3) 연결성 테스트 (ICMP / TCP 포트)


```text

```


```bash

```

예시:
```text

```


---

### 1-5) 열려있는 포트 / 연결 상태 / PID 매핑

```bash
lsof -nP -iTCP -sTCP:LISTEN
```

예시(일부):
```text
Python 2201 user  3u  IPv4 0x... TCP *:8000 (LISTEN)
```

```bash
netstat -anv | grep LISTEN
```

예시:
```text
tcp4 0 0  *.8000                 *.*                    LISTEN
```

PID → 프로세스:
```bash
ps -p 2201 -o pid,comm
```

예시:
```text
  PID COMMAND
 2201 Python
```

### 1-6) 방화벽 상태 / 규칙 확인·추가

## 2) Linux(컨테이너/VM) 네트워크

예시:

### 2-2) 연결성 테스트 (HTTP/TLS/TCP)

```bash

```


예시:

## 5) 실전 트러블슈팅 체크리스트

### 5-1) “포트가 진짜 열렸나?” (로컬/macOS/컨테이너/Pod)

**macOS**
```bash
nc -vz 127.0.0.1 8080
lsof -nP -iTCP:8080 -sTCP:LISTEN
curl -I http://127.0.0.1:8080
```

**Docker**
```bash
docker ps
docker port web
docker logs web
```

**K8s**
```bash
kubectl get pod -o wide
kubectl logs <pod> -n <ns>
kubectl port-forward -n <ns> svc/<svc> 18080:80
```

---

### 5-2) “Pod는 Running인데 서비스가 안 붙는다”

```bash
kubectl get svc -n default
kubectl describe svc api -n default
kubectl get endpoints api -n default
kubectl get pod -n default -o wide
```

확인 포인트:
- Service selector가 Pod label과 매칭되는지
- endpoints가 비어있지 않은지
- targetPort/containerPort가 일치하는지

---

### 5-3) “이미지 Pull 안됨 (ErrImagePull / ImagePullBackOff)”

```bash
kubectl describe pod <pod> -n <ns>
```

Events에서:
- `ErrImagePull`, `ImagePullBackOff`
- 레지스트리 인증 필요하면 imagePullSecret
- 태그 오타/아키텍처 불일치(amd64/arm64)도 빈번

## 6) 원샷 점검 스니펫

### 6-1) macOS: 네트워크 요약 + 포트 리스닝 요약

```bash
echo "== IP CONFIG =="; ifconfig | sed -n '1,80p'
echo
echo "== DNS =="; scutil --dns | sed -n '1,60p'
echo
echo "== ROUTE (default) =="; route -n get default
echo
echo "== LISTEN PORTS (Top 20) =="; lsof -nP -iTCP -sTCP:LISTEN | head -n 20
```

---

### 6-2) Linux(컨테이너/VM): IP/Route/DNS/Listen/Curl 한 번에

```bash
echo "== ip a =="; ip a | sed -n '1,80p'
echo
echo "== ip route =="; ip route
echo
echo "== resolv.conf =="; cat /etc/resolv.conf
echo
echo "== listen ss =="; ss -lntp | head -n 20
echo
echo "== curl https =="; curl -I https://naver.com | head -n 10
```
